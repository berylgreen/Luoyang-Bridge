## 一、选题依据与文化考据

### 1.1 选题背景

泉州洛阳桥（万安桥），始建于北宋皇祐五年（1053年），竣工于嘉祐四年（1059年），由时任泉州知府蔡襄主持建造。它是中国历史上第一座跨海大石桥，也是世界桥梁史上的里程碑式工程。

洛阳桥在建造过程中开创了两项划时代的工程技术：
- **筏型基础**：在江底铺设大面积石堤作为桥墩基础，比西方同类技术早约700年
- **种蛎固基法**：人类历史上首次将生物学原理运用于大型土木工程

2021年7月，"泉州：宋元中国的世界海洋商贸中心"被列入UNESCO世界文化遗产名录，洛阳桥是其22处代表性遗产点之一。

### 1.2 选题理由

本次4C大赛主题为"中国古代建筑成就——中华优秀传统文化系列之六"，限定范围为民居、官府、皇宫、桥梁（1911年以前）。洛阳桥作为古代桥梁工程的巅峰代表，完美契合大赛主题要求。

选择洛阳桥的核心理由：
1. **科学内涵极深**：筏型基础、种蛎固基、船形桥墩、浮运架梁四大技术创新，为信息可视化提供了极为丰富的数据表现维度
2. **数据可量化**：桥梁的几何参数、力学数据、潮汐数据、历代修缮记录等均可精确量化并可视化
3. **文化价值极高**：2021年世界遗产、全国重点文保单位，文化权威性毋庸置疑
4. **地方特色鲜明**：福建泉州本土建筑，体现闽派文化的海洋工程智慧

### 1.3 历史考据来源

本作品所有历史数据均来源于权威学术文献及官方资料：
- 蔡襄《万安桥记》碑文原文（现存洛阳桥南端，国家一级文物）
- 泉州市文物管理委员会《泉州洛阳桥》专著（1985）
- 罗哲文《中国古桥》（1987）
- 唐寰澄《中国科学技术史·桥梁卷》（2000）
- UNESCO世界遗产文献（2021）
- 福建省文物局相关保护规划文件

## 二、信息架构设计

### 2.1 叙事结构

采用**垂直长卷轴**的叙事结构，以滚动驱动的方式引导用户逐步深入了解洛阳桥。全文分为七大篇章：

| 篇章 | 主题 | 核心信息 | 主要可视化形式 |
|------|------|---------|--------------|
| 一 | 序章 | 总览数据速览 | 动态计数卡片 |
| 二 | 造桥者蔡襄 | 历史背景与建造历程 | 交互时间轴 + 贸易柱状图 |
| 三 | 工程奇迹 | 四大技术创新 | 种蛎固基分步动画 + 雷达图 |
| 四 | 潮汐与力学 | 海洋工程力学数据 | 潮汐折线图 + 对比条形图 |
| 五 | 结构解剖 | 六层桥梁结构 | SVG交互截面图 |
| 六 | 文化传承 | 修缮与世遗 | 散点时间图 |
| 七 | 尾声 | 总结与参考文献 | 汇总数据 |

### 2.2 交互设计

1. **滚动触发动画**：采用 Intersection Observer API 监测元素进入视口，实现渐显、滑入等进场效果
2. **数据计数动画**：关键数字以动态递增的方式呈现，增强数据的感染力
3. **图表懒加载**：ECharts 图表在用户滚动到对应位置时才初始化渲染，优化首屏加载性能
4. **桥梁结构交互**：SVG 分层绘制的桥梁截面，点击可高亮对应结构层并显示详细说明
5. **种蛎固基动画**：分四步骤展示生物工程过程，支持前进/后退操作

## 三、视觉设计方案

### 3.1 配色方案

取自宋代美学与海洋工程意象：
- **主色调**：深海靛蓝 — 象征洛阳江的深沉海水
- **强调色**：牡蛎白金 — 象征牡蛎壳的钙化色泽与古典金饰
- **辅助色**：潮水碧 — 象征海洋与生态的清澈
- **石桥灰**：象征花岗岩石材的厚重质感

### 3.2 字体方案

- 标题：Noto Serif SC（思源宋体）— 呼应宋代文化背景的优雅庄重
- 正文：Noto Sans SC（思源黑体）— 保证数据信息的清晰可读

### 3.3 视觉特效

- **玻璃态设计（Glassmorphism）**：信息卡片采用半透明模糊背景，营造层次感
- **波浪动画**：页面中融入潮汐波浪 CSS 动画，呼应海洋工程主题
- **微交互动画**：按钮hover、卡片浮起等微动效提升用户体验
- **视差滚动**：背景元素以不同速度滚动，增强空间深度感

## 四、技术实现说明

### 4.1 技术栈

| 技术 | 用途 | 版本 |
|------|------|------|
| HTML5 | 页面结构与语义化 | - |
| CSS3 | 设计系统、动画、响应式布局 | - |
| JavaScript (ES6+) | 交互逻辑、DOM操作 | - |
| ECharts | 数据可视化图表 | 5.5.0 |

### 4.2 模块化架构

项目采用模块化的文件组织：
- `data.js`：数据层，所有历史数据的单一数据源
- `charts.js`：可视化层，6个ECharts图表的配置
- `scroll-animations.js`：动画层，滚动驱动的动画系统
- `bridge-interactive.js`：交互层，桥梁可视化的SVG与交互逻辑
- `main.js`：控制层，应用初始化与模块协调

### 4.3 性能优化

1. 图表按需懒加载（Intersection Observer）
2. CSS动画使用 `transform` 和 `opacity` 触发GPU加速
3. 滚动事件使用 `passive: true` 和 `requestAnimationFrame` 节流
4. 无框架依赖，整体文件体积极小
