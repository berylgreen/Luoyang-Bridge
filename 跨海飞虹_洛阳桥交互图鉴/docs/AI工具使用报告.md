## 二、各环节AI使用详情

### 2.1 代码开发辅助（DeepSeek）

**使用方式**：利用DeepSeek辅助编写前端交互代码

**典型Prompt示例**：
```
请帮我编写一个使用Intersection Observer API实现的滚动触发动画系统，
当页面元素进入视口时添加CSS动画类名实现渐显效果。
要求：支持延迟触发、支持子元素交错动画。
```

**效果说明**：AI生成的代码框架经过人工审查、修改和调试后集成到项目中。所有最终代码均经过团队成员的理解、测试和优化。

### 2.2 历史文献检索与整理（DeepSeek + Kimi）

**使用方式**：利用AI工具快速检索和归纳洛阳桥相关的历史文献资料

**典型Prompt示例**：
```
请帮我整理泉州洛阳桥自建成以来的历代修缮记录，
以表格形式列出年份、朝代、修缮内容，并标注数据来源的古籍或现代文献名。
```

**效果说明**：AI提供的文献线索经过人工交叉验证，确认所有历史数据的准确性。部分数据通过查阅原始文献（如蔡襄《万安桥记》碑文）进行了二次校对。

### 2.3 设计文档撰写辅助（Kimi）

**使用方式**：利用Kimi辅助组织设计说明书的章节结构和行文表达

**效果说明**：AI提供的文字框架经过团队成员的审阅、修改和补充，确保文档内容的专业性和准确性。

## 三、创意与核心工作声明

### 3.1 原创性声明

本作品的以下核心内容均为团队成员原创：
1. **信息架构设计**：七篇章垂直叙事结构、交互逻辑流程
2. **视觉设计系统**：配色方案、字体方案、组件设计
3. **数据可视化方案**：图表类型选择、数据映射策略、交互方式设计
4. **SVG桥梁截面图**：分层结构的绘制逻辑与交互设计
5. **种蛎固基过程动画**：四阶段分步展示的创意与实现
6. **所有ECharts图表配置**：6个可视化图表的数据选择、配色和呈现方式

### 3.2 AI的辅助角色定位

AI在本项目中的角色严格限定为**"工具"而非"创作者"**：
- AI提供的代码片段和文字素材均经过人工审查、修改、测试
- 所有设计决策（选题、架构、视觉方案）均由团队讨论决定
- 所有历史数据均经过人工交叉验证，不完全依赖AI输出
